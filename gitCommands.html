Git Commands

git reset

Återställer repot till en föregående commit. För att ange vilken commit man vill tillbaka till så skriver man dessutom commithashet.

git diff

Visar ändringar mellan två inputs. Inputs man kan jämföra är t.ex filer, branches, commits osv.

git diff --staged

Samma som --cached. Jämför ändringar i staging area med vad som är i local repo. (Med staging area menas filer/filändringar som är redo att commitas)

git diff branch1...branch2

Jämför ändringar mellan 2 olika branches från och med den sista gemensamma branch. Dvs. om branch1 var main/master och branch2 skapades från den, så hade man bara fått upp alla skillnader i commits från och med när branch2 skapades tills dess senaste commit.

git commit

Pushar alla våra ändringar från staging area till vårt repo i form av en commit. En commit är loggad så att alla ändringar i repot kan spåras.

git commit -a

Samma som --all. Tar alla modifierade/raderade filer automatiskt till staging area så att de kan commitas. Nya filer som inte har lagts till i staging area kommer inte påverkas.

git commit -p

Öppnar ett gränssnitt där du kan manuellt välja vad du vill göra med enskilda ändringar. Detta kommando kommer att gå igenom de ändringar du har gjort tills du har gått igenom dom alla.

git branch

Visar alla tillgängliga branches samt vilken branch du är nuvarande checkad i.

git checkout

Låter dig byta mellan tillgängliga branches genom att ange branchens namn efteråt. Detta låter dig utföra kommandon som utgår från att du är i en specific branch.

git merge

Slår ihop två olika branches till en unifierad commit. Ifall konflikter i filer sker behöver användaren ingripa.

git log

Tar fram repots olika commits samt extra information om commitsen, som t.ex commithashet som kan komma till nytta i bl.a "git reset".

git show

Visar expanderade detaljer till git objekt som t.ex blobs, trees, taggar och commits. Kommandot har enskilt beteende beroende på vilket typ av objekt man tar in.

git rm [file]

Tar bort en fil från både staging area och working directory.

git mv [existing-path] [new-path]

Effektivt raderar en fil från existing-path och skapar en fil med samma innehåll i new-path. Ifall filens namn ändras mellan existing-path och new-path så ändras namnet på filen också.

git remote add

Skapar en remote repository till din lokala repo. Remote repo innebär att repot sitter på en server och att andra utvecklare kan komma åt och koda i den.
